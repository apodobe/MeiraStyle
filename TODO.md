# Точки роста сайта osipovastyle.ru

Дата фиксации: **2026-04-13**

Список потенциальных улучшений (техника, продукт, SEO, визуал, инфраструктура). Приоритеты условные: P0 — критично, P1 — высокий эффект на конверсию/доверие, P2 — заметный выигрыш, P3 — полировка.

---

## P0 — критично

- [x] Не отдавать публично `server.js` и `tmp/` (исходник в `deploy/server.js`, копирование в `dist/` через `postbuild-hosting.mjs`; denylist в Node + правила в `public/.htaccess`).
- [ ] Подключить Google Search Console: задать `PUBLIC_GOOGLE_SITE_VERIFICATION` в `.env` при сборке (мета уже поддерживается в `Layout.astro`).

---

## P1 — продукт и доверие

- [ ] Добавить блок социального доказательства: отзывы, цитаты, имена/роли (школа, шоурум), при возможности — логотипы партнёров.
- [ ] Явно указать географию: Москва / РФ / выезд / онлайн из любой точки.
- [ ] Снять барьер «портфолио по запросу»: PDF по ссылке, кейс на странице или превью материалов без обязательного первого контакта.
- [ ] Дать ценовой ориентир или вилку («от …», «по запросу» с пояснением), чтобы B2B мог оценить порядок до письма.
- [ ] Усилить упоминания Lamoda / Uniqlo: фото с площадок, даты, контекст проекта (без нарушения NDA), чтобы не выглядело как голая декларация.

---

## P2 — производительность и SEO

- [ ] Самохостинг шрифтов (Cormorant Garamond, DM Sans) или оптимизация загрузки Google Fonts (`font-display`, subset), чтобы убрать блокирующий внешний CSS.
- [ ] Адаптивные изображения: `srcset` / несколько размеров WebP для hero и слайдеров (сейчас одна ширина на все вьюпорты).
- [ ] `link rel="preconnect"` к `https://mc.yandex.ru` для ускорения подключения Метрики.
- [ ] Расширить JSON-LD `Person`: `image` (URL героя), `address` / `addressLocality` для локального сигнала.
- [ ] Рассмотреть разметку `Event` / `Course` (или смежные типы), если есть конкретные предложения под сниппеты.
- [ ] Убрать из инициализации Яндекс.Метрики нестандартный параметр `ssr: true` (не описан в документации).
- [ ] Заменить эмодзи ✅ в блоке «Совместное мероприятие» на единый визуальный язык (SVG, номера, акцентная типографика).
- [ ] Блок «Шоурумы и бутики»: добавить кадры или объединить с другим разделом — два слайда выглядят незавершённо.

---

## P3 — UX, визуал, мелочи

- [ ] Страница `404.astro` с брендингом и ссылками на якоря главной.
- [ ] `apple-touch-icon` для закладок на iOS.
- [ ] `<meta name="theme-color">` под палитру сайта (мобильные браузеры).
- [ ] Секция FAQ: аппаратура, сроки подготовки, требования к площадке, запись вебинара и т.п.
- [ ] Подправить hero на мобильных: соотношение сторон портрета vs высота экрана (например `aspect` на брейкпоинтах).
- [ ] Подписи слайдера: `min-h` как временный костыль — при длинных подписях возможен скачок; продумать фиксированную высоту или обрезку с «ещё».
- [ ] Файлы визиток в `public/visitka-*`: если не нужны публично по URL — держать в `export/` или вне деплоя.

---

## Инфраструктура и процесс

- [ ] Перед деплоем: обязательная проверка `npm run build` (скрипт или pre-deploy hook).
- [ ] Документировать/автоматизировать откат при неудачном FTP-загрузке (бэкап предыдущей сборки).
- [ ] При росте частоты правок — CI (сборка + выкладка), preview-окружение по желанию.

---

## Итог

Главный рычаг роста конверсии для B2B-лендинга спикера — **доверие** (отзывы, кейсы, партнёры, география, прозрачность по цене и портфолио). Технический P0 — **безопасность и корректность публичных путей** на хостинге.
